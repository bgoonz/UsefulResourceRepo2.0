Page 19:

Note: You may wish to use the new Connector-J JDBC driver from www.mysql.com. 
Unzip the downloaded file and place the mysql-connector-java-2.0.14-bin.jar file 
in your classpath (your server classpath). To obtain a connection use the 
following code:

Class.forName("com.mysql.jdbc.Driver").newInstance();

Chapter 3

Page 49:

Correction: All references to 'simpleBean' should in fact be 'SimpleBean'. The 
case was changed early on during development of the book and a few references to 
the old name remained after editing. The source code is named 'simpleBean.java' 
and should be changed to 'SimpleBean.java' (as well as the class name itself). 
This will eliminate any further confusion. The same changes can be made to the 
'productBean.java' file.

Page 70 (CustomerBean.java):

Note: It would be a good idea to actually clear all of the JavaBean's attributes 
prior to loading a new set of data. This could be accomplished by moving the 
initialization code out of the constructor and placing it in a separate method 
called initFields(). You could then call this method from both the constructor 
and the loadCustomer() methods.

Page 79:

Correction: Each of the <option> tags are missing a closing bracket. Each tag 
should be something like this (the source code downloaded from the website is correct):

<select name="spouse">
<option value="true" <%= (custBean.getField("spouse", 
FIELD_VALUE).equals("true")) ? "selected" : "" %>>Y</option>
<option value="false" <%= (custBean.getField("spouse", 
FIELD_VALUE).equals("false")) ? "selected" : "" %>>N</option>
</select>

Chapter 4

Note: The case of several source files begin with a lowercase letter. These 
should each be changed to begin with an uppercase letter. The source file names 
as well as the class names themselves need to be modified. For example:

groceryOrder.java --> GroceryOrder.java
groceryItem.java --> GroceryItem.java
simpleTag.java --> SimpleTag.java

Page 97:

Note: The example showing how to nest tags does not work with Tomcat 4.x. The 
code is written correctly to the JSP specifications and actually worked just 
fine with previous versions of Tomcat. With the newer releases of Tomcat, the 
nested tag example does not work correctly.
