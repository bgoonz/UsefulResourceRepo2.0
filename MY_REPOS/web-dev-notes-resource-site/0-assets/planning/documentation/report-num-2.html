Sender Policy Framework

A host's Sender Policy Framework (SPF) describes who can send mail on its behalf. This is done by publishing an SPF record containing a series of rules. Each rule consists of a qualifier followed by a specification of which domains to apply this qualifier to. For more information please see open-spf.org.

Qualifier Mechanism Argument
- (Fail) all

DMARC

DMARC (Domain-based Message Authentication, Reporting and Conformance) is a mechanism for domain owners to indicate how mail purporting to originate from their domain should be authenticated. It builds on SPF and DKIM, providing a method to set policy and to give reporting of failures. For more information please see dmarc.org.

Raw DMARC record:

v=DMARC1; p=none; sp=none; rua=mailto:security@netlify.com, mailto:fexahwip@ag.dmarcian.com; ruf=mailto:fexahwip@fr.dmarcian.com; fo=1
Tag Field Value
p=none Requested handling policy None: no specific action to be taken regarding delivery of messages.
sp=none Requested handling policy for subdomains None: no specific action to be taken regarding delivery of messages.
rua=mailto:security@netlify.com, mailto:fexahwip@ag.dmarcian.com Reporting URI(s) for aggregate data security@netlify.com, Invalid ( mailto:fexahwip@ag.dmarcian.com)
ruf=mailto:fexahwip@fr.dmarcian.com Reporting URI(s) for failure data fexahwip@fr.dmarcian.com
fo=1 Failure reporting options Generate a DMARC failure report if any underlying authentication mechanism failed to produce an aligned "pass" result.
